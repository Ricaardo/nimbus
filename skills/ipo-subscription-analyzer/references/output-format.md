# IPO 分析报告输出格式

> 每次生成 IPO 分析报告时查阅此文件，确保格式一致。

## 核心原则

- **默认产出 MD 文件** — 写入 `reports/ipo_subscription_YYYYMMDD.md`
- **结论先行** — 报告开头 ≤100 字结论，评分+建议+核心原因
- **只保留可操作标的** — 已过暗盘/已上市的标的**不出现在报告中**
- **对比理由后置** — 评分推导、可比对标放在末尾「📎 评分推导」折叠区
- **极简主义** — 只输出核心信息，删除冗余描述
- **表格优先** — 用表格呈现数据，避免长段落
- **控制篇幅** — 正在招股的标的每只 ≤150 字，总报告 ≤1500 字

## 标准报告模板


## 中签率计算格式

每只正在招股的标的必须包含以下计算。甲组抽签结果是 0 手或 1 手，不要写分数手数（如"中签 0.66 手"）：

```markdown
**中签率计算（1 手为单位）**:
- 超购倍数: XXX倍
- 申购金额: XX万（X倍融资）
- 申购手数: XX手（申购金额 / 入场费）
- 单手中签率: ≈ 0.XX% (50% / (超购倍数 × 90%))
- XX手中签概率: ≈ X.X% (1 - (1-单手中签率)^XX手)
- 中签结果: 中签则获 1 手，不中则 0 手（甲组抽签，不可拆分）
- 预期收益 = 中签概率 × 1手收益
```

## 字数控制

| 部分 | 字数上限 |
|------|---------|
| 🎯 结论 | ≤100字 |
| 招股日历 | 表格形式，不限 |
| 每只正在招股标的 | ≤150字 |
| 资金分配方案 | 表格形式，≤100字 |
| 操作清单 | 表格形式，≤50字 |
| 暗盘捞货 | 表格形式，≤100字 |
| 核心风险 | ≤80字 |
| 📎 评分推导 | 表格形式，不限 |
| **总报告** | **≤1500字** |
| 风险提示 | ≤150字 |
| **总报告** | **≤2000字** |

## 不要包含的内容

- **已过暗盘/已上市的标的** — futuapi `is_subscribe_status=False` 的标的直接跳过，不出现
- 长篇业务介绍（1 句话核心亮点即可）
- 详细的评分维度说明（只给出总分和建议等级，明细放 📎 区）
- 冗长的投资逻辑分析（bullet points ≤3 条）
- 重复的风险提示（每个标的 1 个最大风险）
- 过度详细的时间线甘特图（用简单表格）
- 长篇的总结与建议（🎯 结论 ≤100 字）
- 多余的说明文字和过渡段落
- **分数手数表述**（如"中签 0.66 手"、"预期中签 1.2 手"）—— 甲组抽签只能中 0 手或 1 手

## 必须用表格呈现的内容

- 招股日历
- 打新评分排名
- 资金分配方案
- 预期回报测算
- 操作执行清单

## 中文输出规范

- 所有分析报告、摘要、建议使用**中文**输出
- 保留英文：ticker 代码（00700.HK, AAPL）、专有名词首次出现时附英文（基石投资者 Cornerstone Investors）
- 数字格式：使用国际通用格式（1,234.56），货币标注单位（港元/美元/人民币）

## 多市场 Ticker 识别

| 格式 | 示例 | 市场 | 数据源 |
|------|------|------|--------|
| 4-5位数字.HK | 09999.HK | 港股 IPO | **futuapi** → WebSearch |
| 1-5 位大写字母 | RDDT | 美股 IPO | **futuapi** → Finnhub/FMP → WebSearch |
| 6位数字.SH/.SZ | 688xxx.SH | A 股 IPO | AKShare → WebSearch |
| "港股打新" | — | 港股全扫描 | **futuapi** → WebSearch |
| "美股 IPO" | — | 美股全扫描 | **futuapi** → Finnhub/FMP → WebSearch |
| "A股新股" | — | A 股全扫描 | AKShare → WebSearch |
| "今日IPO" | — | 全市场扫描 | 三市场并行（futuapi 优先） |
