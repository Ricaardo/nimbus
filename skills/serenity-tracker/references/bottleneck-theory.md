## 🎙 风格 DNA（识别他的判断特征）

1. **供应链思维，不看终端看卡点**：不直接押 NVDA/终端 AI 应用，押的是**整条 AI 算力供应链上当前最紧的那个环节**。
2. **小盘 + 高赔率 + 早于共识**：偏好市值小、波动大、机构覆盖少、在共识形成前的"卡点纯标的"（pure play），追求不对称上行。
3. **瓶颈会迁移**：核心信念——AI buildout 中，**约束条件(bottleneck)沿供应链移动**；钱在当前/下一个瓶颈处定价权最强的环节。看对迁移节奏 = 超额收益来源。
4. **主题集中**：高度集中在 AI 半导体供应链，不做分散，conviction 标注强弱。

## 🧠 核心框架：瓶颈迁移图（AI 算力供应链）

钱往"需求 >> 供给、且短期扩不出产能"的卡点走。按当前 AI buildout 的卡点链条：

| 环节 | 卡点逻辑 | 代表子领域 |
|---|---|---|
| **算力芯片** | GPU/定制 ASIC（这一层已是大盘共识，非他主战场） | NVDA/AVGO/MRVL（共识区，他更看上游卡点） |
| **互连/光** | 算力堆叠后，**数据搬运**成新瓶颈 → 光互连取代铜 | CPO（共封装光学）/硅光/光模块/InP 衬底/激光器 |
| **存储** | 算力涨→喂数据的**带宽/容量**跟不上 | HBM/存储接口/存储原厂 |
| **衬底/材料** | 光器件上游的**特种衬底**产能极紧 | InP 磷化铟衬底/化合物半导体 |
| **电力/散热** | 数据中心**功耗墙**→供电与液冷成卡点 | 电源/液冷/电网设备 |
| **算力租赁** | 拿不到 GPU 的需求方→**Neocloud** 二房东 | 算力租赁/GPU cloud |
| **太空** | 地面带宽/覆盖瓶颈→**卫星激光通信** | 卫星/星间激光/太空基础设施 |

**选股动作**：① 判断当前最紧的卡点在哪一环；② 在该环找**纯卡点小盘**（业务越纯越好，受益弹性大）；③ 在共识涌入前布局；④ 跟踪瓶颈是否向下一环迁移（迁移=该轮兑现/换仓信号）。

## ⚠ 常见误用 / 风险（务必内化）

- **不能照抄下单**：小盘高波动，进场点/仓位/止损全要自己定；他的"收益%"是自述、未审计。
- **瓶颈可能证伪**：技术路线变了（如 CPO 商用慢于预期）→ 卡点逻辑失效，纯卡点小盘回撤极猛。
- **流动性陷阱**：很多标的日成交极小，冲击成本高、出不来。
- **共识已涌入≠机会**：他的 edge 在"早"；等你听说时若已大涨，不对称性已消失。
- **主题 beta，非个股 alpha**：很多时候是"AI 光通信"整体 beta，单票基本面未必扎实 → 必须叠加 `us-stock-analysis` 做基本面核验。

## 🔗 怎么用（在 nimbus 里）

1. **当 idea 源**：用 `serenity-tracker` 拉他当前看好的卡点方向/标的（实时）。
2. **判断瓶颈位置**：对照上面迁移图，看他押的环节是否=当前真实卡点（交叉 `news-dashboard`/`sector-analyst`）。
3. **必做基本面核验**：候选标的过一遍 `us-stock-analysis` + `valuation`（小盘尤其要查现金流/稀释/流动性）。
4. **登记 thesis**：值得跟的用 `thesis-tracker` 记 conviction + kill criteria（瓶颈迁移走/技术路线变 = kill）。
5. 与价值/宏观大师 lens 互补：他给"在哪个卡点"，价值派给"这家公司值不值"，宏观派给"现在是不是 risk-on 适合小盘"。
